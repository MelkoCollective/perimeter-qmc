Detailed Description

grid class with site_struct as sites

via the defines in conf.hpp one can specified, what grid-type is chosen and how many states the grid has

resets the check flag for all sites in all states

Whenever an operation is performed that concernes the whole grid, visited sites will be flaged as checked. At the end of the operation (e.g. init_loops) one has to clear this flags

void perimeter_rvb::grid_class::clear_tile_spin ( )
inline

reset the spin-checked-flags on the tiles

After a spinupdate, the tiles have to be checked again if they are now or still updateable. Clearing this bit will not check for that, but enable the check if a tile is visited

void perimeter_rvb::grid_class::copy_to_ket ( )
inline

copies spins from bra to ket

In case of no shift, the ket_spins are the same as the corresponding bra_spins. If a shift is set, it will permute the spins accordingly. It's just a trick for a nicer implementation, so that the two_bond_update doesn't have to "jump" between layers (states)
