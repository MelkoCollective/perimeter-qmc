it uses boost cpu_timer s.t. it only measures the usertime, not the walltime. for now one has to link libboost_timer.so and libboost_system.so to the binary

)

autotuned low-overhead progress display

Parameters
iis the loop, variable that is assumed to be in the intervall [0-workload) call this function in the loop that corresponds to the parameter i. It will print every 1-2s the progress in % and an expected remaining duration (constant workload assumed). It achives this behavior using an power 2 modulo operation (binary &) at the start of the function. The modulo-factor is changed dynamically, s.t. the print only happens every 1-2s. The overhead is minimal since it's only a binary & operation most of the time. (takes 20% of the time of a single mersenne-rng() call)
